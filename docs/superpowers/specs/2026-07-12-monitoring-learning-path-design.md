# 前端监控与埋点进阶教学设计

## 一、学习目标

本课程以当前 `Monitoring&EventTracking` 项目为唯一实践载体，目标是让学习者从“能借助 AI 写 Vue 页面”提升到能够独立解释、设计、实现和验证前端监控与埋点系统。

“达到大厂水平”在本课程中具体指：

- 能讲清浏览器监控信号的来源、触发时机和限制。
- 能设计稳定、可扩展的事件模型与 SDK 模块边界。
- 能实现错误、性能、行为和自定义埋点的核心采集链路。
- 能处理批量上报、失败重试、生命周期补偿、去重、采样和 SDK 自保护。
- 能说明数据质量、隐私合规、版本治理、Source Map、指标聚合和告警的生产实践。
- 能脱离 AI，独立完成关键模块的小型实现并排查常见问题。

普通 Vue 页面开发只服务于制造监控场景和展示监控结果，不作为课程主线。

## 二、当前项目判断

当前项目已经具备一个教学版最小闭环：

1. Vue 页面制造错误、路由、点击和性能场景。
2. `collectors` 监听浏览器或路由信号。
3. `track` 构造统一事件并补充公共上下文。
4. 队列按数量或时间批量上报。
5. Express 服务在内存中接收事件。
6. 事件中心查询并展示结果。

当前实现还不是生产级 SDK。主要缺口包括：

- 性能采集只有基础 Navigation Timing，没有 Web Vitals、长任务和资源性能。
- 队列发送失败会丢失已经出队的数据，也没有并发控制、重试和页面卸载补偿。
- 采集器缺少卸载机制、重复初始化保护和统一的安全执行边界。
- 事件协议缺少 SDK 版本、环境、页面标识、用户标识策略和协议版本。
- 点击埋点过于宽泛，缺少声明式埋点、字段约束、隐私过滤和数据治理。
- 缺少自动化测试、数据质量验证、采样策略、Source Map 还原和告警设计。

## 三、教学方式

采用“先闭环、后深挖”的方式。

每个模块统一按以下节奏推进：

1. 先说明该模块解决的业务问题。
2. 阅读当前代码并手动画出数据流。
3. 找出教学版实现与生产级要求之间的差距。
4. 让学习者先预测代码行为或完成一个小片段。
5. 一起修改代码并运行真实场景验证。
6. 用面试问题和故障案例复盘。

AI 可以协助搭建和审查，但关键函数会要求学习者先口述思路或补全局部代码，避免只会复制结果。

## 四、课程阶段

### 阶段 1：快速跑通现有 Demo

- 建立全局架构图和事件数据流。
- 启动前端与服务端，验证完整上报闭环。
- 从当前打开的 `performance.ts` 理解采集器的基本结构。
- 认识 `collector -> track -> queue -> sender -> server -> event center` 的职责边界。

完成标准：能够不看代码讲清一次性能事件如何到达事件中心。

### 阶段 2：监控核心能力

- JavaScript 运行时错误、Promise 异常、资源加载错误。
- Vue 框架错误与接口错误的边界。
- Navigation Timing、Paint Timing、Web Vitals、Long Task 和资源性能。
- 路由访问、停留时长、点击行为与自定义业务埋点。

完成标准：能够为一种新信号设计采集器，并接入统一事件管道。

### 阶段 3：SDK 工程化

- 配置、生命周期、重复初始化和销毁。
- 事件协议、上下文、版本和类型设计。
- 队列、批量、节流、去重、采样、失败重试和离开页面补偿。
- SDK 自保护、性能开销、循环上报防护和测试。

完成标准：能够解释并实现可靠传输与安全失败，不让监控影响业务。

### 阶段 4：埋点治理与生产实践

- 代码埋点、声明式埋点、可视化埋点的取舍。
- 事件命名、字段字典、Schema 校验、版本兼容和数据质量。
- 用户与会话标识、脱敏、隐私同意和敏感字段过滤。
- Source Map、聚合指纹、指标口径、告警阈值和监控平台分层。

完成标准：能够写出一份可供多人团队执行的埋点方案，并识别错误的数据口径。

### 阶段 5：作品化验收

- 补齐可重复演示的场景和事件筛选能力。
- 完成关键纯逻辑模块的自动化测试。
- 形成架构说明、事件协议、运行说明和生产化差距清单。
- 进行一次从需求到埋点设计、实现、验证和复盘的综合练习。

完成标准：能够独立演示项目，并回答设计选择、故障处理和扩展方案。

## 五、系统边界与数据流

项目分为四层：

- 场景层：Vue 页面负责产生可观察行为。
- SDK 层：采集器、核心事件模型、上下文和传输层。
- 接收层：Express 接收并暂存事件。
- 展示层：事件中心查询并呈现事件。

主数据流为：

```text
浏览器/Vue/Router 信号
  -> Collector
  -> track() 统一建模与补充上下文
  -> Queue 批量与可靠性策略
  -> Sender HTTP/Beacon 上报
  -> Express 接收与校验
  -> Event Center 查询与展示
```

任何新增能力都应归入明确层次，不能把采集、传输和页面展示揉在同一个文件中。

## 六、错误处理与验证原则

- 监控代码失败不能破坏业务主流程。
- 发送失败不能静默丢失关键批次，必须有明确策略和上限。
- 采集的数据必须能在浏览器开发者工具、服务端和事件中心三处交叉验证。
- 纯逻辑模块优先自动化测试；浏览器 API 采集器使用单元测试与真实浏览器联调结合。
- 每完成一个模块，都要验证正常路径、异常路径、重复触发和边界条件。

## 七、范围控制

课程不会偏移到通用 Vue 页面教学、复杂 UI 设计或完整后端平台建设。需要讲解的 Vue、TypeScript、HTTP 和浏览器知识，只围绕监控与埋点主线展开。

本阶段不会建设真实数据库、海量数据计算平台或完整告警平台；这些内容会讲清架构位置、接口和生产取舍，但 Demo 保持可理解、可运行和可验证。

## 八、成功判定

最终不是以“代码文件写完”为完成标准，而是同时满足：

- Demo 可以稳定运行和重复演示。
- 学习者能独立画出架构与数据流。
- 学习者能解释核心浏览器 API 和事件指标含义。
- 学习者能独立增加一个采集器或业务埋点。
- 学习者能发现并修复至少一种上报可靠性问题。
- 学习者能给出生产环境中的采样、隐私、治理和告警方案。
